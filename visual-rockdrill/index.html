Guided Slice Design Exercise

Rock Drill Table

Rock Drill Table is the name of this exercise. Use its 7-step design drill to shape a slice before coding. Fill in the behavior, slice order, roles, seams, refusal lists, proof steps, and bypass risks. The page turns that into three copyable artifacts.

Persist + Bootstrap

Turn this drill into a real workshop project

Your draft auto-saves in this browser while you work. When the drill is ready, download the drill JSON, run the bootstrap command, and the workshop can create the mission directory, first slice, and tmux session for you.

What this page does

This is a guided rehearsal before coding. You define the behavior, break it into slices, name the parts of the system, connect them with messages, then turn that drill into exportable project files.

Workshop Use

Bootstrap Setup

Set the repo name and acceptance harness before you export. This turns the drill into a concrete starter packet for the next project.

Station 1

Mission

One sentence, behavior in -> behavior out. Name the level of work before you reach for implementation.

Station 2

Slices

This is the shipping order, not the idea list. Every slice names the risk it retires and the end-to-end proof it keeps green.

Station 3

Draw

Use the training symbols to name the parts of the system. Roles are not free prose; they are explicit artifact parts with plain meanings.
